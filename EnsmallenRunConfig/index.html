    node_list_comment_symbol string 0..1 The comment symbol to skip lines in the nodes file. Lines starting with this symbol will be skipped. .
    default_node_type string 0..1 The node type to be used when the node type for a given node in the node file is None. .
    nodes_column_number integer 0..1 The number of the column of the node file from where to load the node names. .

    minimum_node_id integer 0..1 The minimum node ID to be used, when loading the node IDs as numerical. .
    numeric_node_ids boolean 0..1 Whether to load the numeric node IDs as numeric. .
    node_list_numeric_node_type_ids boolean 0..1 Whether to load the node types IDs in the node file to be numeric. .
    skip_node_types_if_unavailable boolean 0..1 Whether to skip the node types without raising an error if these are unavailable. .
    load_node_list_in_parallel boolean 0..1 Whether to load the node list in parallel. When loading in parallel, without node IDs, the nodes may not be loaded in a deterministic order. .
    edge_type_path string 0..1 The path to the file with the unique edge type names. .
    edge_types_column_number integer 0..1 The number of the column of the edge types file from where to load the edge types. .
    edge_types_column string 0..1 The name of the column of the edge types file from where to load the edge types. .
    edge_types_number integer 0..1 The number of the unique edge types. This will be used in order to allocate the correct size for the data structure. .
    numeric_edge_type_ids boolean 0..1 Whether the edge type names should be loaded as numeric values, i.e. casted from string to a numeric representation. .
    minimum_edge_type_id integer 0..1 The minimum edge type ID to be used when using numeric edge type IDs. .
    edge_type_list_separator string 0..1 The separator to use for the edge type list. Note that, if None is provided, one will be attempted to be detected automatically between ';', ',', tab or space. .
    edge_type_list_header boolean 0..1 Whether the edge type file has an header. .
    edge_type_list_support_balanced_quotes boolean 0..1 Whether to support balanced quotes while reading the edge type list. .
    edge_type_list_rows_to_skip integer 0..1 Number of rows to skip in the edge type list file. .
    edge_type_list_is_correct boolean 0..1 Whether the edge types file can be assumed to be correct, i.e. does not have something wrong in it. If this parameter is passed as true on a malformed file, the constructor will crash. .
    edge_type_list_max_rows_number integer 0..1 The maximum number of lines to be loaded from the edge types file. .
    edge_type_list_comment_symbol string 0..1 The comment symbol to skip lines in the edge types file. Lines starting with this symbol will be skipped. .
    load_edge_type_list_in_parallel boolean 0..1 Whether to load the edge type list in parallel. When loading in parallel, without edge type IDs, the edge types may not be loaded in a deterministic order. .
    edge_path string 0..1 The path to the file with the edge list. .
    edge_list_separator string 0..1 The separator to use for the edge list. Note that, if None is provided, one will be attempted to be detected automatically between ';', ',', tab or space. .
    edge_list_header boolean 0..1 Whether the edges file has an header. .
    edge_list_support_balanced_quotes boolean 0..1 Whether to support balanced quotes while reading the edge list. .
    edge_list_rows_to_skip integer 0..1 Number of rows to skip in the edge list file. .
    sources_column_number integer 0..1 The number of the column of the edges file from where to load the source nodes. .
    sources_column string 0..1 The name of the column of the edges file from where to load the source nodes. .
    destinations_column_number integer 0..1 The number of the column of the edges file from where to load the destinaton nodes. .
    destinations_column string 0..1 The name of the column of the edges file from where to load the destinaton nodes. .
    edge_list_edge_types_column_number integer 0..1 The number of the column of the edges file from where to load the edge types. .
    edge_list_edge_types_column string 0..1 The name of the column of the edges file from where to load the edge types. .
    default_edge_type string 0..1 The edge type to be used when the edge type for a given edge in the edge file is None. .
    weights_column_number integer 0..1 The number of the column of the edges file from where to load the edge weights. .
    weights_column string 0..1 The name of the column of the edges file from where to load the edge weights. .
    default_weight float 0..1 The edge weight to be used when the edge weight for a given edge in the edge file is None. .
    edge_ids_column string 0..1 The name of the column of the edges file from where to load the edge IDs. .
    edge_ids_column_number integer 0..1 The number of the column of the edges file from where to load the edge IDs. .
    edge_list_numeric_edge_type_ids boolean 0..1 Whether to load the edge type IDs as numeric from the edge list. .
    edge_list_numeric_node_ids boolean 0..1 Whether to load the edge node IDs as numeric from the edge list. .
    skip_weights_if_unavailable boolean 0..1 Whether to skip the weights without raising an error if these are unavailable. .
    skip_edge_types_if_unavailable boolean 0..1 Whether to skip the edge types without raising an error if these are unavailable. .
    edge_list_is_complete boolean 0..1 Whether to consider the edge list as complete, i.e. the edges are presented in both directions when loading an undirected graph. .
    edge_list_may_contain_duplicates boolean 0..1 Whether the edge list may contain duplicates. If the edge list surely DOES NOT contain duplicates, a validation step may be skipped. By default, it is assumed that the edge list may contain duplicates. .
    edge_list_is_sorted boolean 0..1 Whether the edge list is sorted. Note that a sorted edge list has the minimal memory peak, but requires the nodes number and the edges number. .
    edge_list_is_correct boolean 0..1 Whether the edges file can be assumed to be correct, i.e. does not have something wrong in it. If this parameter is passed as true on a malformed file, the constructor will crash. .
    edge_list_max_rows_number integer 0..1 The maximum number of lines to be loaded from the edges file. .
    edge_list_comment_symbol string 0..1 The comment symbol to skip lines in the edges file. Lines starting with this symbol will be skipped. .
    edges_number integer 0..1 The expected number of edges. Note that this must be the EXACT number of edges in the graph. .
    load_edge_list_in_parallel boolean 0..1 Whether to load the edge list in parallel. Note that, if the edge IDs indices are not given, it is NOT possible to load a sorted edge list. Similarly, when loading in parallel, without edge IDs, the edges may not be loaded in a deterministic order. .
    verbose boolean 0..1 Whether to show a loading bar while reading the files. Note that, if parallel loading is enabled, loading bars will not be showed because they are a synchronization bottleneck. .
    may_have_singletons boolean 0..1 Whether the graph may be expected to have singleton nodes. If it is said that it surely DOES NOT have any, it will allow for some speedups and lower mempry peaks. .
    may_have_singleton_with_selfloops boolean 0..1 Whether the graph may be expected to have singleton nodes with selfloops. If it is said that it surely DOES NOT have any, it will allow for some speedups and lower mempry peaks. .
    directed boolean 0..1 Whether to load the graph as directed or undirected. .
    name string 0..1 None .
